# Git 历史秘密清理审批 Runbook

此操作会重写提交哈希并影响所有协作者，不能由自动实现步骤擅自执行。

## 前置审批

- 仓库所有者确认需要从所有 refs 清理的精确路径和模式。
- 安全负责人确认相关密钥已撤销/轮换；历史清理不作为撤销手段。
- 发布负责人冻结 push、PR 合并和镜像同步，通知所有 fork/clone 所有人。
- 创建受控镜像备份并记录清理前 commit、tag 和分支清单。

## 建议流程

1. 在隔离的 mirror clone 中使用 `git filter-repo --path keys/rsa_private.der --invert-paths`；不要直接在工作仓库试运行。
2. 扫描全部 refs、tag、LFS、release asset 和 CI artifact，确认目标材料不存在。
3. 由第二人复核 diff、tag 映射、构建与测试结果。
4. 在维护窗口强制推送经批准 refs，轮换所有可能受影响的凭据。
5. 要求协作者删除旧 clone 后重新克隆；关闭或清理旧 fork、缓存与镜像。
6. 保存审批、命令、哈希映射和扫描证据，不保存被清理秘密本身。

本项目当前未执行历史重写，也未删除用户的算法样例文件。
